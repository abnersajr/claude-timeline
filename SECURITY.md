# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in **claude-timeline**, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please use one of the following methods:

1. **GitHub Security Advisory (preferred):** Create a private security advisory via the [Security tab](https://github.com/abnersajr/claude-timeline/security/advisories/new) of the repository. This keeps the report private and allows coordinated disclosure.

2. **Email:** Send a detailed report to **abnersajr@users.noreply.github.com**.

### What to Include

Please include as much of the following as possible in your report:

- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue or a proof-of-concept.
- The version(s) of claude-timeline affected.
- Any suggested fix or mitigation, if applicable.

## Response

We aim to acknowledge receipt of your report within **48 hours**. We will work with you to understand and address the issue before any public disclosure.

## Scope

The following are **in scope** for security reports:

- The **claude-timeline CLI tool**.
- The **claude-timeline web UI**.

The following are **out of scope**:

- **Claude Code** itself (report those to Anthropic directly).
- The **Anthropic API** (report those to Anthropic directly).
- Third-party dependencies unrelated to this project's functionality.

## Supported Versions

We provide security fixes for the **latest release only**. Please ensure you are running the most recent version before reporting.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Disclosure

We are committed to responsible disclosure. We will coordinate with you on the timing of any public disclosure once a fix is available.
